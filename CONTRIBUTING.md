# Contributing Guide
Thanks for helping improve this trusted producer resource hub.

## Before You Submit
Read:
- [CURATION_STANDARD.md](CURATION_STANDARD.md)
- [COMMUNITY_SUBMISSIONS.md](COMMUNITY_SUBMISSIONS.md)

## What to Contribute
- Official plugin and vendor resources
- Legitimate sample/sound resources with clear licensing
- Technical learning links for audio, DSP, and production engineering
- Broken link reports and quality corrections

## Submission Requirements
Each submission must include:
- Resource name
- Official URL
- Category (plugin, sample, learning, community, toolchain)
- Why it is useful for producers
- License/pricing notes when relevant

## Pull Request Guidelines
- Keep each PR focused on one logical change
- Use concise, factual descriptions
- Avoid hype/marketing language
- Prefer official source links over secondary mirrors
- Do not include personal referral/affiliate links

## Style
- Keep list entries short and readable
- Use consistent capitalization and punctuation
- Remove dead/redirect-heavy links where possible

## Review Standard
If a source cannot be verified, it should not be merged.
