# Community Submissions
Use this document when proposing additions to the list.

## Preferred Route
Open a GitHub issue using:
- `Resource submission`
- `Broken link report`

## Submission Format
- Name:
- URL:
- Category:
- Why it belongs:
- License/pricing notes:

## Fast Review Tips
- Use official links
- Keep descriptions factual and short
- Include enough context to verify quickly

## Decline Reasons
- Unverified source
- Broken or region-locked pages with no alternatives
- Duplicate resource with weaker quality
